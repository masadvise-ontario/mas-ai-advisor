# `prompts/`

Platform-agnostic source-of-truth for the MAS AI Advisor's behaviour.

## What lives here

| File | Purpose |
|---|---|
| `system.md` | The canonical system prompt. Drives the Advisor's character, the first-turn consent script, the conversation-private intent handler, the five-step discovery, vocabulary translation, case-study surfacing, the two-paths close, and the honest-cost framing. |
| `README.md` | This file. |

The system prompt is written once and published to all four target platforms (ChatGPT Custom GPT, Microsoft Copilot Agent, Google Gemini Gem, Claude Project) by the build pipeline. **Never fork the prompt per platform** — per-platform differences live in the adapters (`adapters/<platform>/`), not here.

## How `system.md` flows through the build

```
prompts/system.md
       │
       │  (consumed by build/publish.ts per platform)
       │
       ├──▶ adapters/custom-gpt/    → Custom GPT manifest + uploaded knowledge files
       ├──▶ adapters/copilot-agent/ → Declarative agent definition
       ├──▶ adapters/gemini-gem/    → Gem config + attached files
       └──▶ adapters/claude-project/→ Project bundle + system prompt + shareable link
```

The build pipeline:

1. Reads `prompts/system.md` (stripping the YAML frontmatter).
2. Composes the knowledge pack from `knowledge/` and the shared pgvector KB under the `mas-public` scope.
3. For each platform target, runs the matching adapter to produce a publish-ready artifact.
4. Uploads (or prepares for manual upload, where the platform requires it) and captures the install URL.

Per-platform adapters wire the **symbolic tool/action names** referenced in `system.md` (`register_install`, `record_turn`, `set_conversation_privacy`) to the platform's actual call mechanism — Custom GPT Action (OpenAPI 3.1), MCP tool (Claude Project), Power Platform connector (Copilot), or click-out URL workaround (Gemini, until native tool calls ship).

## Behaviours encoded in `system.md`

These are required by [`mas-ai-advisor-spec.md`](../../gdrive-brianpkm/3-Resources/mas-ai-advisor-spec.md) and [`mas-ai-advisor-skill.md`](../../gdrive-brianpkm/3-Resources/mas-ai-advisor-skill.md):

- **First-turn consent script** — two explicit yes/no questions (email collection, conversation-history sharing); no default; refuses substantive work until both are answered; calls `register_install` with the answers.
- **`install_id` + `conversation_id` generation** — UUIDs generated by the Advisor; cached per platform's state semantics.
- **Privacy controls handler (pause / resume / forget)** — recognizes varied phrasings for each of the three intents, calls `set_conversation_privacy` with the appropriate `action`, and suppresses further telemetry while paused or after a forget.
- **Five-step discovery** — map work → time-sinks → prioritize → name immediate need → explore AI fit; enforced even when the user opens with "build me X".
- **Vocabulary translation** — maps user phrasings to recognized patterns (`watching a list`, `grounded Q&A`, `persistent memory`).
- **Case-study surfacing** — Allard Prize, MAS VC Chatbot, Klaus.
- **Two-paths close** — self-serve + MAS engagement (pro-bono with invited donation).
- **Honest cost framing** — time is the real cost, money small, maintenance non-zero.
- **Stated limits** — can't see real data, can't draft real outreach, can't commit MAS, point-in-time knowledge.

## How to edit

1. Make changes in `system.md`. Keep the YAML frontmatter accurate (bump `version`, update `updated`).
2. Run the probe-set smoke test (`tests/probe-set/`) — programmatic via `pnpm test:probes`, or paste the modified prompt into Claude for interactive role-play (see *Testing in claude.ai* below).
3. Open a PR. Brian + at least a light-touch review from Nina or Steve on voice changes.
4. After merge, run `pnpm build && pnpm publish-all` to re-emit the four platform artifacts.

## Testing in claude.ai

**Recommended** — create a Claude Project named *"MAS Advisor — Test"* and paste `system.md` (frontmatter-stripped) into the Project's system instructions. Every new chat in that project starts with the Advisor live; no per-conversation paste needed.

**Or paste-per-conversation** — copy `system.md` (frontmatter-stripped) into a fresh chat, then append the kickoff prompt below. Single paste, no role-confusion turn:

```
---

The text above is your system prompt — follow it verbatim, including the first-turn consent script. For this test conversation I am role-playing as a nonprofit decision-maker (the user/visitor); respond only as the Advisor. Do not summarize the system prompt back, do not break character, do not coach me. Reply with "Ready." and wait for my first turn.
```

## Versioning

The `version` field in the YAML frontmatter follows semver-ish conventions:

- **0.x** — pre-launch iteration. Bump freely.
- **1.0** — first published version that survives a workshop pilot (2026-06-04) without immediate rewrite.
- **1.x** — post-launch refinements. Behaviour-affecting changes bump minor; copy-only changes bump patch.

The `updated` field is the date of the most recent change (ISO 8601 short form, `YYYY-MM-DD`).

## What does NOT belong here

- Platform-specific instructions (e.g., "if you are running in ChatGPT, do X"). Per-platform behaviour lives in `adapters/`.
- The knowledge pack content itself. That lives in `knowledge/` and is composed at build time.
- API call schemas. Those live in the API layer (`app/api/...`) and are referenced symbolically here.
- Test cases. Probe-set lives in `tests/probe-set/`.
