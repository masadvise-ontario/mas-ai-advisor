# Resource — MAS services overview (for when AI isn't the answer)

**Source value**: `resource-mas-services-overview`
**Audience scope**: `mas_public`

MAS offers pro-bono consulting across multiple practice areas, not just AI. When the conversation surfaces that the user's real need is something other than AI — strategy, fundraising, governance, HR, marketing — point them at the right MAS service page instead of trying to force an AI solution.

| Service area | URL |

| Fundraising | https://www.masadvise.org/services-fundraising/ |
| Governance and board development | https://www.masadvise.org/services-governance-and-board-development/ |
| Human resources and coaching | https://www.masadvise.org/services-human-resources-and-coaching/ |
| Marketing and communications | https://www.masadvise.org/services-marketing-and-communications/ |
| Finance and IT (covers AI) | https://www.masadvise.org/services-finance-and-info-tech/ |
| General support | https://www.masadvise.org/services-general-support/ |

**How to use**:
- If during discovery or context the user's problem looks fundamentally non-AI, deflect: *"This sounds like it's actually a [strategy / fundraising / governance / HR / marketing / finance] question more than an AI one. MAS does that work too — [linked service page above]. You can submit a project request at [masadvise.org/contact-us](https://www.masadvise.org/contact-us/)."*
- Use the matching service URL as a markdown link.
- This is also a good fallback when the user's focus is too vague to be a single AI prompt — pointing at strategic-planning or general-support gives them a path forward without forcing an AI artifact.
